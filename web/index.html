Every matching event, delivered.

No polling. No chain indexer to operate. No glue code per integration. Sluice replaces ~600 lines of bespoke event-listener plumbing with one signed deploy.

Any MCP client drives Sluice in natural language.

MCP is an open standard, so this is not a Claude-only trick. One hosted URL plugs Sluice into Claude, Cursor, Windsurf, Cline, VS Code, or anything that speaks MCP. The hosted endpoint needs no key and serves two tools, four resources and two prompts: read matched deliveries and fire a sandbox event. Subscribing and cancelling sign a transaction, so those two tools live in the stdio server you install locally. Five tools in total.

Terminal · one command 
claude mcp add --transport http sluice https://sluice.unitynodes.com/mcp

Speed is half the product. Getting the event to you is the other half.

RETRIES
Three attempts with backoff (1s, 4s, 16s) before a POST is marked failed.
SIGNED
HMAC-SHA256 on every POST, so you can reject anything not from Sluice.
IDEMPOTENT
A stable key per delivery, so a retry never double-counts on your side.
REPLAY
Re-send recent deliveries on demand, with no new on-chain cost.

On the roadmap, and honestly not in v0.1: a durable at-least-once queue that survives a matcher restart and auto-redelivers once your endpoint recovers. That gap is the line between a demo and production, and we would rather name it than hide it.

What's not.

Sluice is testnet-only and intentionally small. You'll hit limits. They're documented, not hidden. Roadmap is public; vote with usage.
