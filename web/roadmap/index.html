From a testnet prototype to a mainnet service.

Where Sluice goes next, how it pays for itself, and what a grant would fund. Honest about what is shipped and what is not.

Now
LIVE ON TESTNET
  • On-chain escrow contract. SubscriptionRegistry (Rust + Odra 2.8): create_subscription, record_delivery, top_up, cancel_subscription, with four emitted events.
  • Off-chain matcher reading the live CSPR.cloud stream, median end-to-end ~140 ms on testnet.
  • Three delivery channels from one subscription: HMAC-signed webhook, MCP tool call, live WebSocket.
  • MCP server, stdio (5 tools) and hosted Streamable-HTTP (2 read-only tools; signing stays local), 4 resources, 2 prompts, for any MCP client.
  • Predicate language: JSON, AND with nested OR, 12 operators, plain-English parser (rule-based, no LLM).
  • Every delivery billed on-chain via record_delivery. Web workspace, CLI, self-host installer, Docker, Prometheus + Grafana, TypeScript and Python clients.
Q3 2026
NEXT
  • Wallet-native subscribe via TransactionV1. Create and top up a subscription from a browser wallet, no CLI or key file.
  • Historical replay. Backfill matches from CSPR.cloud so a new subscription can be seeded against past events, and a matcher restart no longer drops history.
  • SSE delivery channel. A lighter alternative to WebSocket for front ends behind restrictive proxies.
  • Rust SDK. A first-class client alongside the TypeScript and Python ones.
Q4 2026
MAINNET
  • Mainnet contract. Production deployment of SubscriptionRegistry with the redesigned TransactionV1 flow.
  • Free tier for entry. A no-cost daily allowance so developers can build against mainnet before paying, the way Alchemy and Helius win adoption. Self-host stays free; prepaid escrow is only the hosted, high-volume path.
  • Volume pricing tiers. A per-delivery rate that steps down as monthly volume grows.
  • Dead-letter queue. Failed deliveries after retry exhaustion land in a durable queue with a replay endpoint, so no match is silently lost.
  • x402 metered tier. An optional pay-per-delivery path for callers who prefer metered billing over prepaid escrow.
2027
LATER
  • Cross-contract predicates. Match arbitrary contract events, so DeFi and RWA protocols can subscribe to their own domain events.
  • Timelocked subscriptions. Activate or expire at a block height or timestamp, for scheduled and campaign-bounded monitoring.

Free to try and self-host. Pay per delivery on the hosted mainnet tier.

Testnet
Free. No charge to try, subscribe, or self-host against testnet.
Mainnet, hosted
Per-delivery CSPR escrow. Lock up front, each delivery deducts a small fee via record_delivery. Auditable on cspr.live, refundable on cancel. Volume tiers lower the rate as usage grows.
x402 metered
An optional pay-per-delivery path via x402 for callers who prefer metered billing over prepaid escrow.
Self-host
Free and open source (MIT). Run your own matcher against the same contract. The contract is the source of truth, so self-hosting is first-class, not a downgrade.

The hosted service earns per delivery. Self-hosting keeps the ecosystem honest and gives large operators an escape hatch. Both paths generate on-chain record_delivery activity on Casper.

Why this matters for Casper

Sluice lowers the bar for building agentic and event-driven apps on Casper from a multi-week indexer project to a single MCP install. It is the reaction layer the AI Toolkit was missing, and it composes with the rest of the toolkit instead of competing: it reads from CSPR.cloud, pairs with x402 for pay-per-delivery, and hands events to CSPR.trade-driven agents. Every active subscription is a continuous stream of fee-paying record_delivery transactions, so the service scales on-chain usage as it grows.
